Overview

Welcome to the Agent Orchestrator API documentation. This comprehensive guide covers all endpoints for managing AI agents, task queues, cron jobs, organizations, team management, billing, and more.

Base URLs:
Development: http://localhost:18765/api
Production: https://api.formatho.com/api

Key Features

  • 🤖 Agent Management - Create, configure, and control AI agents
  • ✅ TODO Queue - Priority-based task management
  • ⏰ Cron Scheduling - Automated recurring tasks
  • 🏢 Organizations - Multi-tenant architecture
  • 👥 Team Management - Invite members and manage permissions
  • 💾 State Persistence - Save and restore agent state
  • 💳 Stripe Billing - Subscription and payment management
  • 📊 Analytics - Conversion funnel tracking
  • 🔄 WebSocket - Real-time updates

API Version

Current version: v1.0.0

Authentication

Note: Authentication is currently in development. Most endpoints support both authenticated and public access. Organization-based filtering requires proper authentication headers.

Bearer Token Authentication

Most API endpoints require authentication using a Bearer token in the Authorization header:

Authorization: Bearer <your-api-key>

Getting Your API Key

  1. Sign up or log in to your Agent Orchestrator account
  2. Navigate to Settings → API Keys
  3. Generate a new API key
  4. Store it securely - you won't be able to see it again

Go

package main

import (
    "bytes"
    "encoding/json"
    "net/http"
)

type LoginRequest struct {
    Email    string `json:"email"`
    Password string `json:"password"`
}

func login() (string, error) {
    reqBody := LoginRequest{
        Email:    "user@example.com",
        Password: "your-password",
    }
    
    body, _ := json.Marshal(reqBody)
    resp, err := http.Post(
        "http://localhost:18765/api/auth/login",
        "application/json",
        bytes.NewBuffer(body),
    )
    if err != nil {
        return "", err
    }
    defer resp.Body.Close()
    
    var data map[string]interface{}
    json.NewDecoder(resp.Body).Decode(&data)
    
    return data["access_token"].(string), nil
}

Quick Start Guide

1. Create an Agent

const response = await fetch('http://localhost:18765/api/agents', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': 'Bearer YOUR_API_KEY'
  },
  body: JSON.stringify({
    name: 'My First Agent',
    provider: 'openai',
    model: 'gpt-4o',
    system_prompt: 'You are a helpful assistant.',
    work_dir: '~/sandbox'
  })
});

const agent = await response.json();
console.log('Created agent:', agent.id);

2. Create a TODO Task

import requests

todo = requests.post(
    'http://localhost:18765/api/todos',
    headers={'Authorization': f'Bearer {api_key}'},
    json={
        'title': 'Analyze sales data',
        'description': 'Process Q4 sales CSV and generate insights',
        'priority': 5,
        'agent_id': agent_id,
        'skills': ['python', 'data-analysis']
    }
).json()

print(f'Created TODO: {todo["id"]}')

3. Start the Task

resp, err := http.Post(
    fmt.Sprintf("http://localhost:18765/api/todos/%s/start", todoID),
    "application/json",
    nil,
)

4. Monitor Progress via WebSocket

const ws = new WebSocket('ws://localhost:18765/ws');

ws.onmessage = (event) => {
  const data = JSON.parse(event.data);
  console.log('Update:', data);
  
  if (data.type === 'todo_status' && data.todo_id === todoId) {
    console.log(`Task ${data.todo_id} is now ${data.status}`);
  }

Agents

Agents are AI-powered workers that execute tasks. Each agent has its own configuration, system prompt, and work directory.

Agent States

  • idle - Agent is ready to accept tasks
  • running - Agent is actively executing a task
  • paused - Agent execution is paused
  • stopped - Agent has been stopped
  • error - Agent encountered an error

TODOs

TODOs are tasks that agents execute. They support priority levels, dependencies, and skill requirements.

TODO States

  • pending - Task is waiting to be queued
  • queued - Task is in the execution queue
  • running - Task is being executed
  • completed - Task finished successfully
  • failed - Task encountered an error
  • cancelled - Task was cancelled

    timezone: 'America/New_York',
    agent_id: 'agent-uuid',
    task_name: 'backup',
    task_config: {
      target: '/backups/daily',
      retention_days: 30
    }
  })
}).then(r => r.json());

Cron Syntax Reference

┌───────────── minute (0 - 59)
│ ┌───────────── hour (0 - 23)
│ │ ┌───────────── day of month (1 - 31)
│ │ │ ┌───────────── month (1 - 12)
│ │ │ │ ┌───────────── day of week (0 - 6) (Sunday = 0)
│ │ │ │ │
* * * * *

Common Examples

  • 0 0 * * * - Every day at midnight
  • 0 */2 * * * - Every 2 hours
  • 0 9 * * 1 - Every Monday at 9 AM
  • 0 0 1 * * - First day of every month

Python Example

import websocket
import json

def on_message(ws, message):
    data = json.loads(message)
    
    if data['type'] == 'agent_status':
        print(f"Agent {data['agent_id']} is now {data['status']}")
    elif data['type'] == 'todo_status':
        print(f"TODO {data['todo_id']}: {data['status']} ({data.get('progress', 0)}%)")

ws = websocket.WebSocketApp(
    'ws://localhost:18765/ws',
    on_message=on_message
)

ws.run_forever()

Go Example

package main

import (
    "encoding/json"
    "log"
    "github.com/gorilla/websocket"
)

func main() {
    conn, _, err := websocket.DefaultDialer.Dial("ws://localhost:18765/ws", nil)
    if err != nil {
        log.Fatal(err)
    }
    defer conn.Close()

    for {
        _, message, err := conn.ReadMessage()
        if err != nil {
            log.Println("read:", err)
            return
        }

        var data map[string]interface{}
        json.Unmarshal(message, &data)

        switch data["type"] {
        case "agent_status":
            log.Printf("Agent %s: %s", data["agent_id"], data["status"])
        case "todo_status":
            log.Printf("TODO %s: %s", data["todo_id"], data["status"])
        }
    }
}

Error Handling

Error Response Format

All errors follow a consistent format:

{
  "error": "Description of what went wrong"
}

HTTP Status Codes

Code Description
200 OK - Request successful
201 Created - Resource created successfully
204 No Content - Success with no response body
400 Bad Request - Invalid input or missing required field
401 Unauthorized - Authentication required
403 Forbidden - Insufficient permissions
404 Not Found - Resource doesn't exist
500 Internal Server Error - Server error

Common Error Messages

  • "name is required" - Missing required field
  • "organization not found" - Invalid organization ID
  • "agent not found" - Invalid agent ID
  • "todo not found" - Invalid TODO ID
  • "cron job not found" - Invalid cron job ID
  • "unauthorized" - Missing or invalid authentication
  • "permission denied" - User lacks required permissions

Error Handling Example

try {
  const response = await fetch('http://localhost:18765/api/agents', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': 'Bearer YOUR_API_KEY'
    },
    body: JSON.stringify({ name: 'Test Agent' })
  });

  if (!response.ok) {
    const error = await response.json();
    throw new Error(error.error || 'Request failed');
  }

  const agent = await response.json();
  console.log('Created agent:', agent);
} catch (error) {
  console.error('Failed to create agent:', error.message);
}
