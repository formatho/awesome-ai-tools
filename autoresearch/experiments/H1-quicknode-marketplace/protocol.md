# Protocol H1: QuickNode Marketplace Integration

**Experiment ID:** run_001
**Hypothesis ID:** H1
**Created:** 2026-04-19
**Status:** Protocol Locked

## What

Formatho will integrate with QuickNode's marketplace ecosystem to generate customer signups and revenue.

## Prediction

Adding Formatho to QuickNode's marketplace will generate ≥5 signups within 7 days of going live.

## Why

**Rationale:**
1. QuickNode has a structured marketplace with one-click integrations
2. Marketplace provides discoverability to QuickNode's customer base
3. QuickNode serves 82+ chains with 500B+ monthly requests - large TAM
4. Integration is low-friction for QuickNode customers
5. Previous case studies show marketplace partners get traction

**Mechanism:**
- QuickNode marketplace customers browse for tools
- Formatho integration appears in relevant categories
- One-click setup reduces friction
- Trial usage leads to paid conversion

## Success Metric

**Primary:** ≥5 signups within 7 days of marketplace listing going live

**Secondary:**
- Marketplace impressions/views
- Trial-to-paid conversion rate
- Integration usage statistics
- Customer feedback

## Experiment Design

### Phase 1: Preparation (Days 1-2)
1. **Understand QuickNode Marketplace Requirements**
   - Document submission process
   - Identify technical requirements
   - Determine categories and tags
   - Check approval timeline

2. **Prepare Formatho Integration Package**
   - Create integration documentation
   - Build SDK/API connectors if needed
   - Prepare demo/example code
   - Create marketplace assets (screenshots, description)
   - Set up onboarding flow

3. **Design Integration User Journey**
   - QuickNode customer clicks "Add" button
   - OAuth/authorization flow
   - Dashboard integration
   - First-use experience

### Phase 2: Submission & Approval (Days 3-7)
1. **Submit to QuickNode Marketplace**
   - Complete application form
   - Upload integration package
   - Provide documentation
   - Submit for review

2. **Follow Up on Approval**
   - Track application status
   - Respond to any requests
   - Expedite if possible

3. **Prepare for Launch**
   - Set up analytics tracking
   - Prepare welcome email sequence
   - Configure integration endpoints
   - Test end-to-end flow

### Phase 3: Live & Monitor (Days 8-14)
1. **Go Live**
   - Marketplace listing goes live
   - Announce to relevant channels
   - Track initial impressions

2. **Monitor Daily Metrics**
   - Signups per day
   - Trial activations
   - Integration usage
   - Support inquiries

3. **Adjust as Needed**
   - Optimize listing copy/images
   - Improve onboarding flow
   - Address technical issues

## Baseline

**Current state:** 0 signups from QuickNode marketplace, no integration exists

**Comparison:** Pre-integration baseline (0 signups) vs post-integration (target ≥5 in 7 days)

## Contingencies

1. **Application Rejected**
   - Reason: Document and fix
   - Re-apply with improvements
   - Consider alternative: partner with existing marketplace vendor

2. **Slow Approval Process**
   - Follow up proactively
   - Leverage any QuickNode contacts
   - Parallel: pursue H3 (referral) while waiting

3. **Low Signups After Launch**
   - Analyze marketplace positioning
   - Improve listing copy and screenshots
   - Consider promoted placement options
   - Pivot to H3 (referral) for direct outreach

## Data Collection

**Metrics to track:**
- Marketplace impressions/views (daily)
- Click-through rate to Formatho
- Signups (with source attribution)
- Trial activations
- Paid conversions
- Integration usage (API calls, features used)
- Customer feedback/reviews

**Tools:**
- QuickNode marketplace analytics (if available)
- Formatho internal analytics
- UTM tracking for signups
- Customer feedback forms

## Timeline

- **Day 1-2:** Preparation and documentation
- **Day 3-5:** Submit and wait for approval
- **Day 6-7:** Approval and setup
- **Day 8-14:** Live and monitor (primary measurement period)

**Total experiment duration:** 14 days
**Primary measurement period:** 7 days (after going live)

## Risks

1. **Approval Delay:** Marketplace approval could take longer than expected
   - **Mitigation:** Start H3 (referral) in parallel
   - **Mitigation:** Follow up regularly, leverage contacts

2. **Technical Complexity:** Integration might be more complex than anticipated
   - **Mitigation:** Start with minimum viable integration
   - **Mitigation:** Use QuickNode's SDK and examples

3. **Low Discoverability:** Marketplace listings may not get much traffic
   - **Mitigation:** Optimize for search terms and categories
   - **Mitigation:** Consider paid promotion if available

4. **Revenue Unclear:** Trial users may not convert to paid
   - **Mitigation:** Focus on signups first, optimize conversion later
   - **Mitigation:** Have clear value proposition and pricing

## Research Questions This Will Answer

1. Does QuickNode marketplace generate meaningful discovery traffic?
2. What's the typical approval timeline for marketplace integrations?
3. What categories/listings perform best?
4. How do marketplace users behave vs organic signups?
5. What's the trial-to-paid conversion rate for marketplace-sourced users?

## Dependencies

- QuickNode marketplace submission requirements and process
- Formatho's integration capability and documentation
- Analytics tracking infrastructure
- Developer time for implementation

## Notes

- This is a confirmatory experiment (we're testing our prediction)
- If successful, this validates marketplace partnership model
- If unsuccessful, we learn about discoverability challenges
- Either outcome informs H3 (referral) approach
