# QuickNode Referral Program Analysis

**Date:** 2026-04-19
**Source:** https://www.quicknode.com/referrals
**Purpose:** Understand referral economics for H3 hypothesis

## Referral Program Structure

### How It Works
1. **Sign up** - Create QuickNode account (takes <5 minutes)
2. **Share everywhere** - Use unique referral link to share with builders
3. **Get paid** - Account credited with 50% of referral's first month payment

### Revenue Share Terms

**Commission Rate:** 50% of referral's FIRST month payment only

**Payout Method:** Account credit (applied to QuickNode balance, not cash)

**Payout Timing:** Credits applied when referral pays their first month

### Requirements

- Must be a QuickNode customer to participate
- Referral must sign up using your unique link
- Referral must make a payment for first month
- Terms and conditions apply (https://refer.quicknode.com/terms)

### Tracking

Dashboard access at: https://dashboard.quicknode.com/rewards

Features:
- Unique referral link generation
- Referral status tracking
- Credit balance viewing

## Economic Analysis

### Example Scenarios

**Scenario 1: Small developer sign-up**
- Referral signs up for $49/month plan
- Formatho earns: 50% × $49 = $24.50 account credit
- Can apply credit to own QuickNode usage
- Total potential: Single payout only (first month only)

**Scenario 2: Enterprise customer**
- Referral signs up for $999/month enterprise plan
- Formatho earns: 50% × $999 = $499.50 account credit
- Significant credit if large enterprise deal
- Still only first month

**Scenario 3: Multiple referrals**
- 10 referrals @ $49/month each = 10 × $24.50 = $245 account credit
- Good for reducing infrastructure costs
- Not a recurring revenue stream

## Hypothesis H3 Reassessment

**Original Prediction:** "Setting up a QuickNode referral link will generate ≥$100 revenue within 30 days"

**Revised Analysis:**

**Problem:** This is NOT revenue, it's account credit

**Key distinction:**
- Account credit = reduces own QuickNode costs
- Revenue = cash that can be used elsewhere
- For Formatho, this only matters if we're already using QuickNode

**Hypothesis validity:** 
- If Formatho is a QuickNode customer: This reduces infrastructure costs (value)
- If Formatho is NOT a QuickNode customer: This has no value (can't use credits)

**Revised prediction for H3:**
- If Formatho uses QuickNode: Generate ≥$100 in account credits within 30 days
- If Formatho doesn't use QuickNode: This hypothesis is invalid (no revenue model)

## Strategic Considerations

### Pros
1. **Low friction** - Just create account and get link
2. **High commission** - 50% is generous
3. **No ongoing work** - Set and forget referral link
4. **Reduces costs** - If using QuickNode, this is net savings

### Cons
1. **First month only** - Not recurring revenue stream
2. **Account credit only** - Not cash revenue
3. **Requires QuickNode usage** - Useless if not a QuickNode customer
4. **One-time payout** - Per customer, not ongoing

### Comparison to Other Revenue Models

| Model | Revenue Type | Recurring | Effort |
|--------|--------------|------------|---------|
| Marketplace integration | Cash revenue | Yes (conversions) | High (integration) |
| Referral program | Account credit | No (first month only) | Low (setup link) |
| Startup program credits | Infrastructure savings | Yes (while active) | Medium (application) |
| Enterprise co-selling | Cash revenue | Yes (deals) | High (sales) |
