## Partner Evaluation Framework

### Scoring Criteria (1-5 points each)

**Audience Alignment (max 5)**
- 5: Perfect match - developers building onchain with AI
- 4: Strong match - developers, adjacent use cases
- 3: Good match - technical audience, broader
- 2: Weak match - some overlap
- 1: Poor match - minimal overlap

**Audience Size (max 5)**
- 5: >500k developers
- 4: 100k-500k developers
- 3: 50k-100k developers
- 2: 10k-50k developers
- 1: <10k developers

**Engagement Quality (max 5)**
- 5: Highly active (daily content, strong community)
- 4: Active (weekly content, good engagement)
- 3: Somewhat active (monthly content)
- 2: Low activity (rare content)
- 1: Inactive

**Partnership History (max 5)**
- 5: Many successful case studies, proven model
- 4: Some case studies, open to partnerships
- 3: Few partnerships, unclear willingness
- 2: No visible partnerships
- 1: Known to be unresponsive

**Accessibility (max 5)**
- 5: Easy to contact, fast responses
- 4: Contactable, medium response time
- 3: Contact available, slow responses
- 2: Hard to find contact
- 1: No clear contact path
